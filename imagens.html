Trabalhando com imagens em HTML

A implementação de imagens em HTML é feita de modo simples através da tag <img>, através de seus atributos conseguimos formatar de maneira simples a imagem do modo que ela se adeque corretamente ao nosso site.

Atributos da tag

Como dito anteriormente, esta tag possui alguns atributos que podem ser utilizados, os mais simples são:

  • Atributo width: Define a largura da imagem através de um valor em pixels ou em porcentagem.
  • Atributo height: Define a altura de uma imagem através de um valor em pixel ou em porcentagem.
  • Atributo src: Este atributo define qual imagem será mostrada, ele pode receber arquivos de dentro do projeto ou links de imagens externas, funcionando de maneira semelhante a tags de link.
  • Atributo alt: Este atributo é responsável por uma descrição alternativa da imagem, este atributo é muito utilizado por leitores de tela para descrever imagens para pessoas cegas, além disso, mecanismo de busca usam este atributo para saber do que a imagem se trata

Otimizando o carregamento de imagens

Como visto anteriormente, atributos da tag <img> podem diminuir ou aumentar seu tamanho de acordo com o que for especificado pelo programador, porém esta prática pode deixar seu site lento devido ao "peso" das imagens que forem carregadas. Atualmente isto não é mais um problema tão grande devido a capacidade de transmissão de dados da era atual, porém mesmo com esta capacidade toda, a otimização das imagens ainda se faz necessária para um site ter um carregamento satisfatório.

Entendendo o tamanho das imagens

Arquivos da internet são transmitidos através de pacotes de dados, o tamanho destes pacotes é medido em bytes, a unidade de armazenamento de computadores. Uma página da web nada mais é do que um conjunto de arquivos que foram baixados pelo usuário e estão sendo interpretados pelo navegador, mostrando imagens, textos, icones, vídeos, etc. Então quando colocamos uma imagem em um site, ela será baixada pelo usuário e posteriormente exibida no seu site, o que pode levar a alguns problemas de carregamento se feito de maneira inadequada.

Um exemplo de uso inadequado seria:

Temos uma imagem de tamanho 1920px x 1080px, este é o tamanho de um monitor padrão, porém a imagem será exibida no tamanho de 960px x 540px, se usarmos os atributos width e height conseguimos deixá-la deste tamanho, porém o arquivo original ainda será no tamanho de 1920px x 1080px, resultado em carregamento desncessário de dados, o que gera um carregamento lento da página web.

Para deixarmos nosso site de maneira mais otimizada, podemos fazer o uso de programas que ajustem a qualidade da imagem para que ela seja diminuida e utilize apenas a quantidade necessária de dados, sites como ILoveImg podem fazer compressão de arquivos de imagem para otimizá-las e reduzir seu tamanho, reduzindo também seu peso. Há também softwares como CorelDraw e Photoshop que podem editar as imagens manualmente e ajustá-las exatamente para o tamanho que serão exibidas.

Canais de cor

Canais de cor são canais específicos que cada imagem tem, o padrão utilizado na internet é RGB(Red, Green, Blue) que separa as imagens em 3 canais de cor que são transmitidos aos pixels de uma tela, cada pixel é capaz de emitir os 3 tipos de cor. O valor de cada canal pode ser medido de 0 a 255, onde 0 é totalmente desligado e 255 é o valor máximo que pode ser utilizado, por isso cores RGB possuem 3 valores que se combinados formam uma cor. Um exemplo: A cor preta é formada por todos os canais desligados RGB(0,0,0) e a cor branca é formada por todos os canais no máximo RGB(255,255,255)

Também há o canal alfa, um canal responsável pela transparência em uma imagem, ele possui valores entre 0 e 1, sendo o valor 0,5 uma imagem com 50% de transparência, se tornando possível ver através da imagem.

Existe também a profundidade de bits, ela é necessária para entender como os canais de cores funcionam, ela determina a quantidade de combinações possíveis em um canal RGB, uma profundidade de 8 bits quer dizer que temos 256 variações em cada canal, sendo possível criar até 16.777.216 cores, uma pronfundidade de bits de 16 possui 65.536 valores possíveis para cada canal, sendo possível criar até 281.474.976.710.656 cores diferentes, apesar do olho humano não sera capaz de entender essa quantidade de cores, imagens com o padrão de 16bits possuem uma transição melhor entre cores, principalmente em gradientes.

Há também o formado CMYK(Cyan, Magenta, Yellow, Black), um formato mais utilizado no mundo real por gráficas, pois as combinações de cores são feitas de modo que a totalidade seja a cor preta, por exemplo: A cor branca no CMYK é expresso pelo valor CMYK(0%, 0%, 0%, 0%), já a cor preta é expressa por CMYK(0%, 0%, 0%, 100%) ou por CMYK(100%, 100%, 100%, 100%) para um preto "enriquecido".

Apesar de possuir um aspecto mais opaco em relação a cores RGB, ela é a mais utilizada por gráficas e impressoras devido a como a tinta forma cores. O RBG necesita de um fundo de luz para funcionar, sendo perfeito para telas, que emitem luz e podem misturar as cores apropriadamente, já o CMYK consegue expressar as cores sem esse fundo de luz, que é o caso de materiais como papel, tecidos, etc. Esse "conflito" entre o mundo real e o mundo digital é o motivo pelo qual não conseguimos materializar algumas cores que vemos em monitores, justamente por esses materias não emitirem luz, trabalhando apenas com a reflexão e absorção de raios de luz.

Tipos de imagens

Quando tratamos de imagens, há varios tipos de arquivos que são utilizados, desde arquivos .raw que são extremamente pesados até arquivos .webp que são extremamente leves e já otimzados para o uso na internet, saber quando usar um certo tipo de imagem em seu site pode trazer possibilidades incríveis de estilização, design e claro, velocidade. Abaixo há uma lista com os principais tipos de imagens usadas na web e suas características:

JPG ou JPEG:
É atualmente o tipo de arquivo mais utilizado para imagens num geral, celulares comumente tiram fotos neste arquivo. Apesar de ser grandemente utilizada, ela faz a compactação de imagens com algumas perdas de qualidade se utilizada de maneira muito extrema, além disso, não suporta a transparência em imagens por não ter canal de cores Alfa.
PNG:
Os arquivos .PNG fazem a compactação de imagens sem perdas, sendo muito utilizada quando a fidelidade e qualidade da imagem são o foco, além disso, suportam transparência por terem canal alfa. Por ter um formato de compatação sem perdas e possuirem o canal alfa, imagens .PNG são maiores e mais pesadas do que imagens .JPG, fazendo com que seu uso tenha que ser justificado para não fazer carregamento desnecessário de dados.
GIF:
Formato de imagens animadas, por ter um peso leve para uma animação, possui muito pouca qualidade de imagem e suporte de cores, sendo pouco utilizado atualmente.
SVG:
Formato de imagens para "vetores", imagens que não são necesariamente fotos e são construidas com formas geométricas, possui um tamanho leve e é muito utilizado para logotipos e gráficos vetoriais. Por ser leve e baseada em vetores, não é adequada para fotografias ou imagens detalhadas.
WEBP:
Formato de imagem semelhante ao JPG e PNG, porém com uma capacidade de compactação sem perdas maior que o JPG e com suporte a transparência. Por ser um formato "novo", alguns navegadores mais antigos não possuem suporte a ele.
AVIF:
Um formato de imagem mais moderno que o WEBP, possuindo uma capacidade de compactação maior e mais eficiente, trazendo grande qualidade de imagem com arquivos muito pequenos. Por ser uma tecnologia mais recente que o WEBP, possui muito mais navegadores e dispositivos que não são compatíveis com este tipo de arquivo.
ICO:
Formato de arquivo específico para icones de sites e favicons. Por possuir poucas aplicações se torna um tipo de imagem utilizada apenas em casos muito específicos.
RAW:
Formato de arquivos utilizado apenas em ambito profissional de fotografia, ele não possui limite de cores como a profundidade de bits e outros valores, sendo ideal para a edição de fotos onde se pode trabalhar com diversos tons de cores e iluminação, esse tipo de imagem não é utilizado na web e por dispositivos comuns devido a seu alto peso "sem" necessidade, onde o usuário comum se contentaria com uma imagem JPG. Devido a problemas de compatibilidade, esta imagem provavelmente não será exibida
