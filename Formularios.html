Formulários em HTML

Formulários são extremamente versáteis e utilizados para coleta de informações. Eles podem ser usados para contato, aceitação de termos, orçamentos, entre outros. O HTML nos fornece uma maneira de implementá-los em nosso site com algumas tags específicas. Confira abaixo uma relação das tags utilizadas para criação de formulários:

Tags para criação de formulários

<form>
Esta é a tag que define o formulário. Ela envolve todas as outras tags que criam campos ou configuram o formulário.
<input>
Esta é a principal tag para criação de campos de entrada. Ela é responsável por criar os campos onde iremos inserir informações ou aceitar termos. O atributo type define o tipo de campo. Confira abaixo uma lista com os principais tipos e suas aplicações:
  • Tipo text: Cria uma caixa de texto.
  • Tipo password: Cria uma caixa de texto, mas o texto digitado fica "escondido", como em um campo de senha.
  • Tipo email: Cria uma caixa de texto que aceita apenas endereços de e-mail. Possui uma validação que impede valores inválidos.
  • Tipo submit: Cria um botão para envio do formulário.
  • Tipo radio: Cria botões de seleção única, como em perguntas de múltipla escolha.
  • Tipo checkbox: Cria caixas de seleção independentes, permitindo selecionar mais de uma opção.
  • Tipo number: Cria um campo de texto que aceita apenas números.
<label>
Cria um título ou descrição para um campo. Por exemplo, ao criar um campo para o nome do usuário, esta tag exibe um texto como "Nome:" antes do campo. O atributo for pode ser usado para vincular o rótulo a um campo específico, utilizando o id do campo.
<select>
Cria uma caixa de seleção de opções. Contém outras tags que definem as opções disponíveis.
<option>
Define as opções dentro de uma tag <select>. O atributo value define o valor da opção enviada no formulário. O texto visível ao usuário é definido dentro da própria tag. O atributo selected indica a opção selecionada por padrão, e o atributo disabled impede que a opção seja selecionada.
<textarea>
Cria uma área de texto para inserção de textos maiores, como descrições ou opiniões. Os atributos rows e cols definem, respectivamente, o número de linhas e colunas da área de texto.

Informações complementares

Embora a maioria das tags acima tenha sido bem explicada, o tipo radio precisa de algumas informações adicionais para funcionar corretamente. Além disso, alguns atributos importantes se aplicam a várias tags e devem ser mencionados.

Tipo radio

Como mencionado, o tipo radio cria botões de seleção única. Para que funcionem corretamente, o atributo name é necessário. Ele conecta as opções de um mesmo grupo, permitindo que apenas uma seja selecionada. Por exemplo, em uma questão de prova, todas as alternativas da mesma questão devem ter o mesmo name:

<input type="radio" name="questao_10" value="A"> A <input type="radio" name="questao_10" value="B"> B <input type="radio" name="questao_10" value="C"> C <input type="radio" name="questao_10" value="D"> D

Dessa forma, o usuário pode escolher apenas uma alternativa da questão 10.

Atributos universais

Alguns atributos são indispensáveis para o funcionamento de formulários:

  • value: Define o valor que o campo envia ao formulário. É necessário em campos como radio e option. Também pode ser usado para definir um valor predefinido em campos como text, number e password.
  • id: Atribui um identificador único a uma tag. Este atributo permite redirecionar o usuário a um campo específico ao usar a tag <label>. O id não é exclusivo de formulários e pode ser usado em outros elementos HTML. Vale lembrar que cada id deve ser único na página.
  • required: Este atributo faz com que um determinado campo de um formulário seja de prenchimento obrigatório, não sendo possível enviar o fomulário sem ele
  • placeholder: Apesar de não ser uma atributo que agregue funcionalidades, ele deixa o formulário mais intuitivo adicionando uma legenda aos campos de texto, como se fosse um campo preenchido, mas o usuário não consegue interagir com aquele texto.
