Audio e Vídeo em HTML

Assim como as imagens, o HTML também consegue incorporar arquivos de audio e video em sua página de maneira nativa sem muitas dificuldades, confira abaixo como trabalhar com cada uma dessas mídias.

Trabalhando com audios

Conforme dito anteriormente, o HTML consegue incorporar nativamente este tipo de mídia, usando a tag <audio> conseguimos incorporá-lo corretamente no nosso site definindo um campo de áudio na nossa página. Após definirmos a área do audio, é necessário utilizar mais uma tag dentro do campo para definirmos a mídia que será utilizada, esta tag é a <source>, através do atributo src dela definimos o arquivo que será reproduzido, além disso utilziando o atributo type definimos o tipo de arquivo de audio estamos usando, como mp3, ogg, etc. Confira abaixo um exemplo de código da aplicação e também uma aplicação prática:

Código:

<audio controls>
 <source src="source/audio/audio.mp3" type="audio/mpeg">
</audio>

Trabalhando com vídeos

Apesar de ser um tipo de arquivo diferente, a implementação de vídeos é semelhante a de áudios, apenas com a mudança da tag para a de video que é <video>, usando a mesma estrutura e fazendo uso também da tag <source>. Confira abaixo um exemplo de código e também uma aplicação prática:

Código:

<video controls width="300px">
 <source src="source/video/exemplo-video-html.mp4" type="video/mp4">
</video>

Tag <source> e outros atributos das tags de audio e vídeo

Se repararmos bem, há alguns outros termos escritos tanto na tag <source> quanto nas tags de áudio e video que são extremamente importantes para o funcionamento dessas tags apropriadamente, abaixo há uma lista descrevendo estes atributos e o que eles exatamente fazem em suas tags.

Atributo type:
A tag <source> possui um atributo muito importante que define o tipo de arquivo que estamos passando para a tag de audio ou vídeo, ele é definido por type="audio/tipo de arquivo" ou por type="video/tipo de arquivo". As extenções que o tipo audio suportam são: mpeg(mp3), ogg, wav, webm, aac, flac, x-ms-wma, midi e x-midi. Já as extenções que o tipo de vídeo suportam são: mp4, webm, ogg, x-msvideo, 3gpp, 3gpp2, x-matroska, x-ms-wmv, mpeg e quicktime.
Tendo em vista a quantidade de arquivos suportados, podemos fazer diversas escolhas de como utilizá-los em nosso site, há arquivos que são mais amplamente suportados pelos navegados, arquivos que possuem uma qualidade maior e arquivos que possuem tamanhos menores e auxiliam no carregamento, além disso uma mesma tag de audio ou de vídeo podem conter mais de uma tag <source>, dando mais opções de tipos de arquivos que podem ser reproduzidos por uma mesma tag, aumentando a compatibilidade com os navegadores.

Atributo controls:
O atributo controls é extremamente necessário para a exibição tanto de audios quanto de vídeos, ela habilita os controles padrões de vídeos e audios como play, pause, volume, etc.. Apesar de não ser um atributo obrigatório, ele é extremamente recomendado para ser utilizado, pois ele habilita os controles para o usuário poder interagir com o conteúdo, caso não seja utilizado é recomendado fornecer outros tipos de controles para o usuário poder interagir com o conteúdo, caso não sejam fornecidos e não seja usada nenhuma outro atributo que viabilize a exibição do conteúdo, a mídia não será exibida na página.

Atributo autoplay:
Este atributo é um dos que viabiliza a exibição do conteúdo, ele faz com que o audio ou vídeo seja iniciado automaticamente quando o usuário abre a página, não habilitando controles para o usuário. Alguns navegadores bloqueiam este atributo para melhorar a exeriência do usuário, permitindo que o usuário veja apenas conteúdo que ele ativamente iniciou.

Atributo loop:
Este atributo faz com que a mídia seja tocada em loop, reiniciando toda vez que acaba.

Atributo muted:
Este atributo faz com que o audio ou vídeo seja iniciado com o volume mutado, sem nenhum audio.

Atributo preload:
Este atributo define o modo que a mídea é carregada na página, se o valor for none, o arquivo não será carregado antes de ser reproduzido, se for metadata, carregará apenas os metadados do arquivo, como a duração por exemplo, se for auto, irá precarregar o arquivo completo, este é o valor padrão.

Atributo src:
Apesar da existência da tag <source>, as tags de audio e video também podem receber seus arquivos através do atributo src como as tags de imagem, não sendo necessário usar o atributo type.

Atributos width e height:
Estes atributos são os mesmo utilizados nas tags de imagem, sendo usados nas tags de vídeo para definir a altura e largura que o vídeo será exibido.

Atributo poster
Este atributo define uma capa para o vídeo enquanto ele não é reproduzido, funciona da mesma maneira que uma tumbnail do youtube.

Atributo playsinline:
Este atributo indica que o vídeo deve ser reproduzido em linha (dentro da página) e não em tela cheia, principalmente para dispositivos móveis.

Atributo controllist:
Este atributo define quais controles padrão serão exibidos para o usuário através de alguns parâmetros. O parâmetro nodownload irá remover o botão de download dos controles, o nofullscreen retira o botão para colocar o vídeo em tela cheia, o noremoteplayback desabilita a transmissão da mídea para outros dispositivos, o disablepictureinpicture desabilita o modo picture-in-picture. Para adicionarmos estes parametros, podemos apenas coloca-los um ao lado do outro com apenas um espaço dentro do atributo controllist, exemplo: controlist="nodownload nofullscreen"
